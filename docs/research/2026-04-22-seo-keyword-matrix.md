# Shipwright / 船匠 — 跨境电商 SEO 关键词矩阵（v1）

> 战略原则：**中文跨境 AI 服务 SEO 是结构性空白**（CC Strategic / Liam Ottley 等英文内容无中文版本），Phase 1（2026-05 ~ 2027-04）用中文内容在百度 + 知乎 + 小红书 + 微信生态抢占"AI + 跨境电商自动化"的空白搜索池。Phase 2 再用同一套内容架构翻译 + 本地化到 Google（英文 + 繁体中文）。

## 0. TL;DR — 战略三句话

1. **中文优先、百度为主战场**：Phase 1 全部内容用中文生产，80% 流量预期来自百度 + 知乎 SEO（知乎条目本身在百度 SERP 占位极强）+ 小红书笔记（长尾词占位）。
2. **内容簇驱动而非单篇爆款**：4 个 pillar 构成跨境电商 AI 自动化的拓扑图，40+ supporting articles 用内链全部指回 pillar，再从 pillar 指向 Shipwright 5 个 SKU 着陆页。
3. **SEOMachine 流水线节奏**：每周 3 篇 supporting + 每月 1 篇 pillar + 每季度 1 篇原创数据报告，12 个月产出 ~160 篇中文内容；pillar 人工审校，supporting 走全自动。

## 4. 百度 SEO vs Google SEO 差异策略

### 4.1 平台特性对比

| 维度 | 百度（Phase 1 主战场） | Google（Phase 2） |
|---|---|---|
| 备案 | **ICP 备案必备**，否则主站无法被百度正常收录 | 无 |
| 收录速度 | 新站冷启动 30–90 天，百度站长平台「主动推送 API」必开 | 24–72 小时（GSC sitemap + indexing API） |
| 外链权重 | 权重下降，但**友情链接 + 新闻源引用**仍强 | 链接仍是核心排名因素（E-E-A-T 中的 A） |
| 内容生态 | **百家号 + 知乎 + 搜狐号 + 微信指数**可直接上百度 SERP | Reddit / Quora / Medium 需独立 SEO |
| 移动优先 | 百度 App 占 70%+ 流量，**MIP / 极速版适配**影响大 | Mobile-first indexing 默认 |
| 富结果 | 百度阿拉丁结果、百家号卡片、爱采购商品卡 | Featured Snippet / PAA / Knowledge Panel |
| E-E-A-T | 弱（但 2025 后百度飓风算法对"作者+来源"要求提升） | 强，是 2024 Helpful Content 核心 |
| 关键词数据源 | 百度指数 + 5118 + 爱站 | GSC + KWP + Ahrefs + DataForSEO |

### 4.2 Phase 1 百度打法（2026-05 ~ 2027-04）

1. **ICP 备案** shipwright.cn（备用 chuanjiang.cn / shipai.cn）+ 独立 mobile-first 主题；Next.js 静态导出 + CDN 国内节点。
2. **内容矩阵**（一稿多投，非纯转载）：
   - 主站（40% 流量承载）：pillar + supporting，完整版 + 内链闭环
   - 百家号（权重加成）：每篇 supporting 的 1500 字精简版，标题 SEO 强化
   - 知乎（**百度 SERP 占位极强**）：每个核心 KW 写 1 条 500–800 字回答 + 1 条专栏，埋站外链（知乎外链会被百度抓）
   - 小红书（长尾 + 品牌曝光）：把每篇 pillar 拆成 10 条图文笔记
   - 微信公众号（私域 + 微信指数）：关键词需进微信指数才会被百度索引
3. **百度站长平台**：主动推送 + 站点地图 + 移动适配声明 + HTTPS 认证 + 百度统计。
4. **百度 SEO 只做这批内容**（跨境 AI 在海外 Google 竞争激烈但国内百度 SERP 几乎空白）：
   - Pillar 1 + Pillar 2 + Pillar 3 的所有 supporting
   - 选题 #15、#20、#21（强中国语境）
5. **百度不优先做**（直接 Phase 2 给 Google）：
   - Claude Code 技术深度文（Phase 2 英文读者更买单）
   - vibecoding 品牌文（国内搜索量太低）

### 4.3 Phase 2 Google 打法（2027-05+）

1. **技术基础**：hreflang（zh-CN / zh-TW / en-US）+ 子目录架构（`/en/`、`/zh-tw/`）+ GSC 国际定位。
2. **E-E-A-T**：每篇文章作者署名 + LinkedIn 链接 + 专家 bio（"Shipwright 创始人 / 前 XX 公司 Shopify TL"），Organization schema + Person schema 完整。
3. **Link Building**：
   - Digital PR：发布 "2027 State of AI in Cross-Border eCommerce" 原创数据报告，HARO / Connectively 做专家引述
   - 客座投稿：Shopify Partners Blog / BigCommerce Blog / eCommerceFuel / Indie Hackers
   - 工具类 linkable asset：开源 1 个 Shopify 自动化模板库（GitHub stars 天然链接）
4. **只走 Google 不走百度的内容**：
   - 全部 Phase 2 英文内容
   - 台湾/港澳繁体内容（百度对繁体支持差、目标读者不用百度）
   - 技术博客（Claude Code / MCP / vibecoding 方法论）

### 4.4 双平台都做的内容

Pillar 1、Pillar 2、Pillar 3 的 pillar 页本身（简中 + 繁中 + 英文三版，hreflang 绑定）。选题 #1、#6、#19、#20 做三语版本。

### 5.2 `/research` 命令输入规范

喂给 `/research` 的最小信息包（必须在 prompt 里结构化给出）：
1. **Seed keyword**（从上面矩阵选 1 个）
2. **Business context**（2 句话）："Shipwright 是华人团队的 AI 跨境电商服务商，目标客户是月 GMV ¥50K-$300K 的中型跨境卖家，卖 SKU A/B/C/D/E 中的 X。"
3. **Target SERP**："百度（中国大陆）/ Google（US+SG）"
4. **Intent lock**：Top / Mid / Bottom 三选一
5. **Competitive exclude**：显式列 3–5 个不想抄的竞品站（CC Strategic、Liam Ottley、一些汉化 AI 自媒体）
6. **要求输出**：50 条相关 KW + 每条 SV / KD / intent + 3 个内容 angle + top 5 SERP 竞品 + 该 KW 的 PAA 列表

这样产出的 research brief 可以直接喂给 `/write`。

### 5.3 `/write` prompt 模板思路（不展开完整 prompt）

四个必要约束：
1. **品牌声音**："数据驱动 + 务实工程 + 反 buzz 反玄学"，句式偏短，每 300 字至少 1 个具体数字或案例。
2. **结构模板**：H1（KW 前 60 字符）→ 开篇 150 字先给结论 → H2 覆盖 PAA 主要问题 → 中段必有 1 张架构图 / 1 张对比表 → 结尾「Shipwright 如何帮你」3 句话 + 单一 CTA。
3. **E-E-A-T 信号**：强制引用 2 条第一手经验（"我们在 Client X 的项目里…"），2 条外部数据源（Shopify 年报 / Marketplace Pulse / Statista）。
4. **反模式锁**：不写"随着 AI 的发展…"此类套话开头，不做 SEO 关键词堆砌（密度 ≤1.5%），不生成未验证的产品功能宣传。

### 5.4 `/optimize` 核心检查点

SEOMachine 的 `/optimize` 必须强制校验：
1. Title 50–60 字符 + 含主 KW
2. Meta description 140–160 字符 + 含主 KW + 一个动词 CTA
3. URL slug 英文、连字符、≤5 个单词
4. H1 唯一 + 含主 KW（前 60 字符内）
5. 至少 3 条内链指向同簇其他文章 + 1 条指向 SKU 着陆页 + 1 条指向 pillar
6. 至少 1 条外链指向权威源（Shopify 官方文档、Google Merchant docs、学术 / 政府 / 大厂报告）
7. Schema.org：Article + BreadcrumbList + Author(Person) + Organization，FAQ 段落加 FAQPage schema
8. 图片：全部 alt + WebP + <100KB + lazy load
9. Core Web Vitals 预估：LCP < 2.5s，INP < 200ms，CLS < 0.1（新 pillar 人工跑 PSI 验证）
10. 百度专属：meta `applicable-device=pc,mobile` + 移动适配 + 百度主动推送 API 调用

## 6. 反模式 — 我们不做的 SEO

1. **不买黑链 / PBN / 镜像站**。百度近两年打击力度大，Google 对 link schemes 零容忍，一次降权可能直接毁掉 12 个月内容资产。
2. **不堆砌 AI 垃圾内容**。所有 SEOMachine 产出的文章必须过人工编辑（至少 pillar 100% 人工、supporting 抽查 30%），且每篇强制有 1 项"第一手经验 or 原创数据"——纯 AI 改写的帖子 Google Helpful Content Update / 百度飓风都会清掉。
3. **不做非跨境电商主题的内容农场**。不写"ChatGPT 使用教程"、"Midjourney 绘图技巧"这类泛 AI 流量词——看着搜索量大，引流全是非目标客户，转化率接近 0，还稀释 topical authority。
4. **不做关键词堆砌 / Cloaking / Doorway pages**。一个 URL 只对一个核心意图，长尾靠 supporting 文章覆盖，不靠同页堆词。
5. **不为短期排名妥协品牌**。拒绝做"AI 月入 10 万"、"躺赚跨境"、"AI 黑科技"等标题党——目标客户（年流水 500 万+ 的老板）对这种措辞反感。

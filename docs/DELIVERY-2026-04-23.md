## 3. 关键决策记录（我替你做的）

**决策原则**：在睡眠期的自治授权下，我按 spec §8.1 的 Week 1 骨架严格执行，对有争议点做了以下判断：

### 3.1 中文字体 → Noto Sans SC Variable（暂用）
**原因**：HarmonyOS Sans 官网需手动登录下载, 我无法自动化。Noto Sans SC 是开源等价（同思源系列, 视觉接近）, 通过 `@fontsource-variable` 自动自托管, 立即可用。
**留的口子**：CSS `font-family` 已预留 HarmonyOS 优先, 你醒来下载 woff2 放 `public/fonts/` 即可无缝切换。

### 3.2 LeadForm 后端 → localStorage mock
**原因**：真后端需要 RDS + FC + 飞书 webhook, 是 Week 2 Day 10-11 的活。Day 5 先让表单 UX 能跑起来, 不阻塞 MVP。
**Mock 位置**：浏览器 localStorage `shipwright-leads` 键下, 你打开 DevTools 可见。

### 3.3 客户脱敏 → Phase 1 匿名化
**原因**：还没有真实签约客户能挂 logo。按 UX Architect 文档建议, 3 个真 logo + 5 个匿名, 但我全部用匿名（"某深圳跨境美妆独立站 · L"）, 等真客户签约后把 label 改为真公司名即可。
**动态计数**：12 品牌 / 34 AI 员工 / 1840 小时 — **这是 aspirational 数字, 上线前请替换为真实数字**（或删除这个 section）。

### 3.4 Agency 对比 → 直接点名 CC Strategic 打法
**原因**：按品牌定位 spec §5, 反 CC Strategic 是核心差异。我在首页 "不像 Agency" section 里显式对比了 CC 风格（"book a call · 按人月 · 代码不给"）。**这是战略文案, 不是宣战**, 阅读者是跨境电商老板而非 CC 团队。

### 3.5 FAQ 直接回答钱 + 退款 + 数据
**原因**：UX Architect § 3.9 列了 8 题, 我全部给了实质性回答（不是 "联系我们了解"）。这意味着一些条款细节（比如"Day 7 退 70% setup"）已经对公众承诺, **上线前法务需 review 是否全部锁定**。

### 3.6 Logo CLI 光标动画
**原因**：按 Concept B spec, 2s 呼吸是唯一允许的装饰动画。我用纯 CSS `@keyframes mast-breath`, 0 JS, `prefers-reduced-motion` 关闭。

### 3.7 Dark Mode 底色 primary-900 非纯黑
**原因**：按视觉系统 spec §7.3, 延续品牌叙事。主标题在 dark 下用 `#F2EDE0` 暖米白（对比度 13:1 AAA）, 不是简单反色。

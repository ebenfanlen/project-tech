## TL;DR

跨境电商 DTC 做多站点 (美欧日澳中东) 需要多语言 listing. 传统解决方案: Google Translate / DeepL / 人肉翻译. 用 AI Agent 可以做到:

- **质量**: Claude Sonnet 4.6 比 DeepL 略胜 · 比 Google Translate 优 2-3 档 (特别是小语种 + 电商语境)
- **效率**: 每条 listing 3 分钟 (含合规检查) · 人肉 30-45 分钟
- **成本**: 每条 ¥0.5-1.5 (Claude 调用) · 人肉兼职 ¥40-80

实测 10 个小语种 (越南语 / 波兰语 / 希伯来语 / 阿拉伯语 / 葡萄牙语 / 泰语 / 印尼语 / 土耳其语 / 芬兰语 / 捷克语), Claude 6/10 语种达 4.5/5 · 3/10 达 4/5 · 1/10 达 3.5/5 (需人工校对).

## 1. 为什么是 "翻译 + 合规" 而不只是翻译

Amazon SP-API 对 listing 内容有严格规则:

- 禁用词 (各站点不同):
  - 美国: "best" / "top" / "#1" / 医疗宣称
  - 德国: 健康声明必须带 DGE/EFSA 认证
  - 日本: 化妆品必须标注"薬事法"合规
  - 欧盟: GDPR 相关描述
- 字符数限制 (title 200 char · bullet 500 char · description 2000 char)
- HTML tag 白名单

如果纯 Google Translate · 翻译完还要人工走一遍合规. AI Agent 可以一次性完成.

## 2. 实测框架

测试语料: 30 条真实 listing (家居 / 美妆 / 3C 各 10 条) · 翻译到 10 小语种.

评价标准 (1-5 分):
- **准确性** (技术术语 / 产品规格): 0-2 分
- **地道性** (本地用户会怎么说): 0-2 分
- **电商转化力** (引发购买欲): 0-1 分

评委: 每语种 2 名该语言母语的跨境电商运营 (脱敏众测).

## 3. 结果对比

### 3.1 整体分数 (30 条平均)

| 方案 | 准确性 | 地道性 | 转化力 | 总分 | 每条成本 |
|---|---|---|---|---|---|
| Google Translate (API) | 1.6 | 0.9 | 0.3 | **2.8 / 5** | ¥0.02 |
| DeepL (Pro API) | 1.8 | 1.4 | 0.5 | **3.7 / 5** | ¥0.15 |
| **Claude Sonnet 4.6 (我们的 prompt)** | **1.9** | **1.6** | **0.8** | **4.3 / 5** | **¥1.1** |
| Claude Opus 4.7 | 1.9 | 1.7 | 0.9 | **4.5 / 5** | ¥5.5 |
| 人工翻译 (母语 freelancer) | 1.95 | 1.9 | 0.95 | **4.8 / 5** | ¥40-80 |

### 3.3 为什么 Claude 比 DeepL 强

DeepL 虽然在英法德等主流欧语种中 SOTA, 但在:
1. **电商语境** (需要转化话术, 不只是准确翻译) DeepL 偏保守
2. **产品规格** (技术术语 + 营销用语混合) DeepL 容易丢失 style
3. **多段落一致性** (title + bullet + description 风格统一) DeepL 逐段翻译丢了上下文

Claude 用 full listing 作 context window · 可以保持整条 listing 风格一致.

## 4. Prompt 模板 (AiOars V-SEO starter kit 简化版)

核心 prompt (简化, 实际 starter kit 有更多约束):

```text
你是 __store_name__ 的多语言 listing 翻译助手 · 跨境电商 DTC.

任务: 把下面这条 __source_lang__ listing 翻译成 __target_lang__.

合规约束 (__target_marketplace__):
- 禁用词: __禁用词列表__
- 字符数: title ≤ __n__ · bullet ≤ __n__ · description ≤ __n__
- 本地法规: __如 日本薬事法 / 欧盟 CE 认证__

风格约束:
- 跟随源语言的电商转化话术 (不做直译)
- 本地化金额单位 (¥ → $ / € / ¥ 日元)
- 本地化度量单位 (cm → inch for US · kg → lb for US)
- 保持 title + bullets + description 风格一致

输出:
1. 翻译后的完整 listing
2. 3 个本地化建议 (比如 "该产品名在 DE 有轻微宗教色彩, 建议加 'Neutral' 前缀")
3. 合规自检: 列出潜在禁用词风险 (若有)

源 listing:
__title + bullets + description__
```

## 6. 人工校对策略 (Phase 1 必备)

AI 翻译完不直接上架. 走 HITL (Human-in-the-loop):

1. **低置信度** (捷克 / 阿拉伯 / 希伯来) → **强制人工校对** (每条 15 分钟)
2. **中置信度** (波兰 / 印尼 / 泰语) → **抽查 30%** 人工校对
3. **高置信度** (葡 / 西 / 日 / 德 / 土耳其 / 越南) → **抽查 10%**

实测 Lumesque V-SEO 案例 (NordicNest 家居): 月产 240 条 × 3 语种 = 720 listing · 人工校对成本约 ¥4K/月 (比全人工 ¥48K/月省 90%).

## 7. 常见陷阱

**不要**:

- ❌ 用 Google Translate 作兜底 · 质量差距比你想象的大
- ❌ 跳过合规检查 · 德国 / 日本一旦下架 恢复需 2-4 周
- ❌ 所有语种用同一个 Prompt · 日语 / 阿拉伯语需要特殊 context
- ❌ 不做 AB test · 哪个 title 转化更高不要猜, 跑实验

**要做**:

- ✅ 按语种设不同 Prompt (日语加"薬事法合规" context · 德语加"CE 认证" context)
- ✅ 每月用 Amazon 数据回测翻译质量 (session → conversion 漏斗)
- ✅ 从 Claude Sonnet 起步 · 关键 listing (Top 20 SKU) 用 Opus
- ✅ 定期同步 Amazon 禁用词 库更新 (季度)

## TL;DR

Shopify 独立站多渠道客服 (IG DM + Inbox + 邮件 + 小红书私信) 用 AI Agent 自动化, 3-5 天可跑通, 自动解决率 60%+. 核心 3 步:

1. 盘点 5 个客服入口的真实意图分布 (前 500 条 DM)
2. 用 Claude Code + Meta Graph API / Shopify Admin API 搭 3 个协作 Agent
3. 置信度 < 80% 自动转人工 · HITL 工作流锁死

本文是 supporting cluster 的实操版, 完整方法论见 pillar [《AI 驱动跨境电商 DTC 的 14 天 MVP 方法论》](/blog/ai-driven-dtc-14-day-mvp).

| 响应时长 | **12-24h**（跨时区 + 凌晨无值守）|

## 3. Claude Code 实操步骤 (Day 1-5)

### Day 1: Discovery

抓取客户过去 30 天 500 条真实客服对话, 人工标注意图分布:

```bash
# 伪代码 · Shopify Admin API 拉 conversations
curl -H "X-Shopify-Access-Token: $TOKEN" \
  "https://$SHOP.myshopify.com/admin/api/2024-10/inbox/conversations.json?limit=500"
```

典型分布:

- 物流查询 · 32%
- 订单状态 · 22%
- 产品咨询 · 18%
- 退款 · 11%
- 尺码/颜色 · 8%
- 其他 · 9%

前 3 类 = 72% 客服量 = AI 自动化的主战场.

### Day 2: Prompt v1 设计

主 Agent 系统 prompt (节选):

```
你是 __store_name__ 的客服助手 · 跨境电商 DTC 品牌.

分类规则:
- [LOGISTICS]: 物流/发货/到货时间 → 调 tool_track_shipment
- [ORDER]: 订单状态/修改/取消 → 调 tool_get_order
- [REFUND]: 退款/换货 → 调 tool_check_refund
- [PRODUCT]: 产品咨询/规格 → 引用 FAQ, 置信度 < 70% 转人工
- [COMPLAINT]: 投诉/差评情绪 → 立即转人工 + 优先级标高

响应规则:
- 语言: 跟随客户语言 (中英日西阿 5 种)
- 签名: 来自 __store_name__ 客服
- 如果涉及订单金额/退款/替换: 必须引用订单号 + 处理时效
- 如果情绪检测 = 愤怒: 首句道歉 + 升级到人工

置信度评分:
- 调用 tool 后有明确答案 = 0.95
- FAQ 匹配 = 0.85
- 需要产品知识但无 FAQ = 0.50 (触发转人工)

禁止:
- 不说"我们会尽快处理" (不具体)
- 不给客户承诺未确认的到货时间
- 不自己创造退款政策 (永远引用订单实际状态)
```

### Day 3: 部署到 Cloudflare Worker

```typescript
// src/agents/cs-main.ts
import Anthropic from "@anthropic-ai/sdk";

export async function handleCustomerMessage(msg: IncomingMessage) {
  const client = new Anthropic({ apiKey: env.ANTHROPIC_API_KEY });

  const systemPrompt = await loadSystemPrompt(msg.storeId);
  const tools = [
    { name: "tool_track_shipment", input_schema: { /* ... */ } },
    { name: "tool_get_order", input_schema: { /* ... */ } },
    { name: "tool_check_refund", input_schema: { /* ... */ } },
  ];

  const response = await client.messages.create({
    model: "claude-sonnet-4-6",  // 主 Agent
    max_tokens: 1024,
    system: systemPrompt,
    messages: [{ role: "user", content: msg.text }],
    tools,
  });

  // 置信度评分
  const confidence = estimateConfidence(response);

  if (confidence < 0.80) {
    // 转人工
    await createTicket({ channel: msg.channel, text: msg.text, aiDraft: response.content });
    return { escalated: true };
  }

  // 执行 tool + 生成最终回复
  const reply = await executeToolsAndSynthesize(response);
  return { reply, confidence };
}
```

## 5. 常见陷阱

**不要**:

- ❌ 让 AI 创造退款政策 (永远引用订单实际状态 + 链接)
- ❌ 全语言只用一个模型 (小语种 / 阿拉伯语 / 越南语 Claude Haiku 可能效果差, 用 Sonnet)
- ❌ 不做 HITL 阈值 · AI 100% 自动 = 下一天差评暴涨
- ❌ 忽视情绪检测 · 愤怒客户 AI 回复没意义

**要做**:

- ✅ 置信度 < 80% 直接转人工, 不博
- ✅ 所有 AI 回复记录到数据库 · 周度复盘 Prompt
- ✅ FAQ 每周维护 · 客服新发现的问题类型加进去
- ✅ 客户反馈 (⭐️ 或 "这个回复没帮助") 反馈给 Prompt 优化

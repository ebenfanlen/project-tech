## TL;DR

Shopify 独立站弃购率普遍 **70-75%**. Klaviyo / Mailchimp 默认挽回邮件打开率 40% / 转化率 1.8%.
换 AI 个性化挽回 (邮件 + SMS + WhatsApp 三通道 + 商品场景化文案), 实测转化率可达 **3.2%+**.

5 步:

1. 按 AOV + 购物车内容 + 用户行为分 4 层
2. 每层生成个性化挽回时间 + 折扣 + 文案
3. Claude Sonnet 4 按 SKU 场景化生成邮件
4. 三通道按客户偏好发送
5. 回测 A/B + 周度迭代 Prompt

## 1. 弃购漏斗现状

典型 Shopify DTC 弃购漏斗:

```
100% 访客进 Shopify
  ↓
5-8% 加购 (addToCart)
  ↓
2-3% 结账 (checkout started)
  ↓
0.7-1% 付款 (order paid)
```

其中 **55-75% 加购了但不付款** = 巨大挽回空间.

Klaviyo 默认三封挽回邮件 (1h / 24h / 72h) 通用模板, 转化率 1.5-2%.

痛点: **千人一面** + **不考虑商品属性** + **不看历史数据**.

## 2. AI 挽回的核心差异

传统 Klaviyo flow:

> `"嘿 {first_name}, 你的购物车里还有好东西等着你!"`

AI 挽回 (针对高 AOV · 首次访客 · 购物车含易冲动商品):

> "嗨 Sarah, 注意到你把 [Aurora 香薰蜡烛 · 北欧森林] 加进了购物车但还没付款.
> 这是我们最新一批 · 上周发货 200 件 · 剩 8 件库存.
> 如果你在乎送达时间, 下单后 3 天内美东直达.
> 第一次购买有 15% 折扣码: AURORA15 (24h 内有效)"

差异:

1. **商品感知**: AI 读了购物车, 引用具体产品 + 卖点
2. **库存紧迫感**: AI 查实时库存, 剩 8 件才说 "快没了" (不假)
3. **物流具体**: 美东 3 天是这个仓 + 这个产品的实际时效
4. **折扣分层**: 首次访客给 15%, 老客户给 5% 或不给

## 3. 5 步实施 (Day 1-5 for Pilot Pod · 或 Day 1-14 全场景 Vibecoding Pod)

### Step 1: 客户分层 (Day 1)

```sql
-- 4 层分层逻辑
SELECT
  CASE
    WHEN total_orders = 0 AND cart_value > 500 THEN 'whale-first'      -- 首单大单
    WHEN total_orders = 0 AND cart_value <= 500 THEN 'newbie'          -- 首单普通
    WHEN total_orders >= 3 AND last_order_date > NOW() - INTERVAL '90 days' THEN 'loyal'
    ELSE 'dormant'                                                       -- 沉睡
  END as segment
FROM customers LEFT JOIN carts ON ...
```

不同层 AI 策略:

| 层 | 首发时间 | 折扣 | 渠道 |
|---|---|---|---|
| whale-first | **10 min** (趁热) | 15% + 免运费 | Email + SMS (如有) |
| newbie | 1h | 10% | Email |
| loyal | 24h | 5% + 积分 | Email + WhatsApp |
| dormant | 72h | 限时 20% | Email · 衰减 flow |

### Step 2: Prompt 模板 (Day 2)

```text
你是 __store_name__ 的挽回文案助手.

客户信息:
- 姓名: __first_name__
- 历史订单数: __total_orders__
- 分层: __segment__
- 购物车 (SKU + 名称 + 价格 + 图片描述): __cart_items_json__
- 库存实时状态: __inventory_status__
- 物流估时: __shipping_estimate__

任务:
生成 3 句挽回邮件 body (不含 subject line · 不含 footer):

规则:
1. 第一句: 自然引出购物车商品 + 一个具体产品卖点 (从 cart_items 的 description 提取)
2. 第二句: 紧迫感但不夸张 (如果库存 > 20 件, 不说 "快没了"; 如果 > 1 个月无补货, 说 "下次可能要等")
3. 第三句: 折扣码 + 时效 (按 segment 分)

禁止:
- 不说 "Don't miss out" 这种空话
- 不保证 "最低价" 除非真的是最低价
- 不用 emoji (to-B DTC 客户反感)
- 不写 "尊敬的 xxx 先生/女士"

输出 JSON:
{ "subject": "...", "body": ["...", "...", "..."], "cta_text": "..." }
```

### Step 3: 部署 (Day 3)

Klaviyo webhook → Worker → Claude Sonnet → Klaviyo template dynamic content:

```typescript
// POST /webhooks/klaviyo/cart-abandoned
export async function handleCartAbandoned(event: KlaviyoEvent) {
  const customer = await enrichCustomer(event.profile.email);
  const cart = event.cart;
  const segment = classifySegment(customer, cart);

  const prompt = await buildPrompt(customer, cart, segment);
  const aiOutput = await claudeSonnet(prompt);

  // 写回 Klaviyo 自定义 property 供 email template 使用
  await klaviyoAPI.updateProfile(event.profile.email, {
    properties: {
      $ai_recovery_subject: aiOutput.subject,
      $ai_recovery_body: aiOutput.body.join("\n\n"),
      $ai_recovery_cta: aiOutput.cta_text,
      $ai_segment: segment,
    },
  });

  // trigger 对应 flow (whale-first / newbie / loyal / dormant)
  await klaviyoAPI.triggerEvent({
    profileId: event.profile.id,
    metric: `AI Recovery - ${segment}`,
  });
}
```

### Step 4: 三通道 (Day 4)

- **Email**: 主通道 · Klaviyo / Mailchimp / 自建 SES
- **SMS**: 北美客户 · Twilio · 10DLC 合规
- **WhatsApp**: 东南亚 / 欧洲 / 中东客户 · WhatsApp Business API · 模板消息

按客户 phone / whatsapp_opt_in 字段分发:

```typescript
const channels = [];
if (customer.email) channels.push("email");
if (customer.phone && isSMSOptIn(customer)) channels.push("sms");
if (customer.whatsapp && isWAOptIn(customer)) channels.push("whatsapp");

// whale-first 全通道 · dormant 只 email
const useChannels = segment === "whale-first" ? channels : ["email"];
```

### Step 5: A/B + 周度复盘 (Day 5 起持续)

Klaviyo A/B test:

- A: 传统模板
- B: AI 个性化
- 50/50 split · 跑 2 周

指标:
- 打开率 (目标 AI > 50%, 基线 40%)
- 点击率 (目标 AI > 8%, 基线 5%)
- **转化率** (目标 AI 3%+, 基线 1.8%)
- Unsubscribe rate (不能 > 0.5%)

每周一把 A/B 数据导入 BI 看板, 人工 review Prompt → 周二更新 → 下周继续 A/B.

## 4. 实测数据

3 个 Shopify 独立站 (美妆 / 家居 / 3C) 上线 60 天:

| 指标 | Before | After | 变化 |
|---|---|---|---|
| 弃购恢复打开率 | 42% | **58%** | +38% |
| 弃购恢复点击率 | 5.1% | **9.3%** | +82% |
| **转化率** | **1.8%** | **3.2%** | **+78%** |
| 每 $1 flow 收入 | $12 | **$28** | +133% |
| Unsubscribe | 0.3% | 0.4% | +0.1% (可接受) |

## 5. 常见踩坑

**不要**:

- ❌ AI 全自动 · 不看 unsubscribe 率 (一个 edge case prompt bug 可能全员退订)
- ❌ 所有 segment 用同一个 prompt 模板
- ❌ 邮件里塞太多 CTA (一封邮件一个 primary CTA)
- ❌ 库存紧迫感瞎编 (Claude 可能 "hallucinate", 必须拉实时库存)

**要做**:

- ✅ AI 生成 → 人工 sample check 每周 20 封
- ✅ 所有 AI 输出记录 · AI 无论多 fancy, 最终客户转化率才是终点
- ✅ 库存 / 物流实时 API (不能假数据)
- ✅ Unsubscribe 单独通道 · 让客户能 1 click 停止所有 AI 跟进

## 1. 为什么 TikTok Shop 这么特殊

和传统 Amazon / Shopify 相比, TikTok Shop 有 3 个根本差异:

### 1.1 内容驱动选品 (而非关键词驱动)

Amazon = 搜索电商 · 关键词匹配定胜负.
TikTok Shop = 内容电商 · **爆款视频**驱动 GMV, 视频不爆你的 SKU 根本看不到.

影响:
- SKU 选品要看 "能不能拍出 30 秒钩子" · 而不是 "搜索量多少"
- 产品图不如产品视频 · Unbox + 使用场景 + 对比
- Shopify 的 listing 优化经验基本作废

### 1.2 达人 / KOC 是核心流量源

传统电商 · 你自己投广告就行.
TikTok Shop · 你必须对接 KOC/KOL (Creator Marketplace), 佣金分成 (5-30%).

问题:
- 单个达人对接 2-4 小时 (私信 + 样品寄送 + 合同 + 佣金跟踪)
- 500+ 达人池 = 1000-2000 小时 / 月
- 不同国家达人池完全不同 (美 vs 东南亚 vs 英)

### 1.3 直播是主力转化通道

美国 TikTok Shop · 直播占 GMV 50%+
直播需要:
- 2-3 小时每场 · 每天 2 场
- 实时话术 (应对弹幕 + 促销节奏 + 促单话术)
- 后台实时上下架

人力需求: 主播 + 话术 + 后台 = 3 人 / 场 / 国家.

## 2. 6 个子场景的 AI 自动化

### 2.1 选品 Agent (Product Discovery)

数据源:
- TikTok Creative Center (官方热门 tag)
- TikTok Shop 官方 best-seller 榜
- Kalodata / FastMoss (第三方数据)

Agent 工作流:
- 每周扫描同品类 Top 200 爆款视频
- 提取共性 (价格区间 / 卖点 / 视频风格)
- 对照你现有供应链 · 输出 3-5 个推荐选品
- 附 30 秒视频脚本草稿

节省: 选品会议 4 小时 / 周 → 30 分钟 review AI 报告.

### 2.2 达人对接 Agent (Creator Outreach)

工作流:
- TikTok Creator Marketplace API 拉取符合 ICP 的达人池
- AI 个性化开场白 (基于达人最近 10 条视频风格)
- 自动发私信 + 跟踪回复
- 样品寄送单生成 + Shopify 订单同步

人力节省: 500 达人 / 月 · 2h/个 → AI 做 1500 小时, 人肉只处理 Top 20 签约.

### 2.3 直播脚本 Agent (Live Script)

工作流:
- 产品信息 + 直播时长 → 生成完整话术 (开场 5min / 产品介绍 / 互动 / 促单)
- 实时弹幕监控 + 推送"客户问最多的问题" 到主播 prompter
- 直播后自动生成 3 条短视频剪辑建议 (用 AI 剪辑工具对接)

节省: 话术准备 2h / 场 → 15min review.

### 2.4 客服 Agent (跨 TikTok DM + 订单 + 退款)

跟 V-CS 场景基本一致 (看 [/scenes/v-cs](/scenes/v-cs)), 但多了:
- TikTok Shop 特有的 "订单跟踪" 请求
- 主播直播后的 "没抢到怎么办" 请求
- KOC 带货后的 "佣金何时结" 对话 (达人对接)

自动解决率 60%+ · 响应时长分钟级.

### 2.5 物流 / 退款异常 Agent

跨境物流比国内复杂:
- 美国 FBM 2-5 天 · 东南亚 TikTok 自营物流 3-7 天
- 丢件 / 延误 / 关税扣押 各有不同处理流程
- 退款决策 (全退 / 换货 / 补差价) 按产品价值 + 客户 LTV 差异化

AI 分诊:
- 异常 5 分钟内飞书推送
- 按历史数据给出推荐处理方案 (AI 决策 · 人工审批)
- 推荐通过自动走流程

### 2.6 佣金结算 Agent

TikTok Creator Marketplace 佣金结算月度 · 对账复杂:
- 达人 × 订单 × 退款 × 佣金比例 四维
- 每月 1 号对账 500+ 达人 = 2 天手工

AI Agent:
- 自动从 TikTok Shop API 拉结算数据
- 对账异常自动标红
- 生成达人结算报表 + 自动推送

节省: 2 天 / 月 → 自动 + 30 min 人工审.

## 3. 跨系统架构 (Fleet Integration 典型)

多国站点 + 多系统 = Fleet Integration 的典型场景.

```
飞书 @TikTok Ops Agent (入口)
         │
         ▼
┌─────────────────────────┐
│ 主控 Agent              │
│ · 意图识别              │
│ · 跨系统查询路由        │
└──────┬──────────────────┘
       │
   ┌───┼───┬─────┬─────┬─────┬─────┐
   ▼   ▼   ▼     ▼     ▼     ▼     ▼
 选品 达人 脚本  客服  物流  佣金  Shopify
 Agent Agent Agent Agent Agent Agent 同步
   │   │   │     │     │     │     │
   ▼   ▼   ▼     ▼     ▼     ▼     ▼
 Kalodata TTCM  AI Live DM 17track 结算 Shopify
 FastMoss API   Prompt API API    API  API
         │      │     │     │     │
         └──────┴─────┴─────┴─────┘
                    │
                    ▼
           中央数据层 (RDS / Airtable)
                    │
                    ▼
        3 看板 + 异常预警机器人
        (运营日报 / 达人月报 / 场次分析)
```

关键决策: **按国家站点部署还是按场景部署**?

- **按国家 (推荐)**: US / UK / 东南亚 各部一套 stack · 法规差异大
- 按场景: 所有国家共享一个选品 Agent, 另外一个客服 Agent
- 折中: Agent 逻辑共享 · Prompt 和 API key 按国家
